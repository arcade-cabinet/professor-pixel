# Pillar 1 — Frontend

> The React + Vite shell. Everything the user sees runs here.
> Other pillars (runtime, grading, lessons) plug into this one through `@lib/*` imports.

## Stack

| Layer | Choice |
|-------|--------|
| Framework | React 18 (functional components only — no class components) |
| Language | TypeScript 5.6, `strict: true` |
| Build | Vite 5 |
| Routing | wouter |
| Async / cache | TanStack Query 5 |
| UI primitives | Radix + shadcn/ui (`app/components/ui/`) |
| Styling | Tailwind CSS 3 (theme tokens via CSS variables) |
| Motion | Framer Motion |
| Code editor | Monaco (loaded from CDN at runtime) |
| Form validation | react-hook-form + Zod |

## Layout

Capacitor-style flat: one `app/`, one `src/`, one `public/` at the repo root.

```text
app/                            React TSX (.tsx only)
  components/{editor,pixel,pygame,wizard,ui}/
  pages/                        Top-level routes; `_dev/` for dev-only screens
  assets/pixel/                 Bundled mascot portraits — imported as @assets/pixel/*

src/                            Non-TSX TypeScript (.ts), domain-decomposed
  assets/ errors/ grading/ hooks/ lessons/ monitoring/ net/ python/
  storage/ types/ utils/ wizard/
  pygame/{components,runtime,templates}/
  types/pyodide.d.ts            Ambient declarations for Pyodide globals

public/                         Static assets, served as-is
  assets/catalog.json           Generated by scripts/build-asset-catalog.mjs
  pyodide/                      Vendored Pyodide runtime (T2.3, gitignored)
  api/static/lessons.json       Lesson content (validated by Zod at fetch time)
```

## Aliases

Configured in `vite.config.ts` and `tsconfig.json`:

| Alias | Resolves to | What goes here |
|-------|-------------|----------------|
| `@/*` | `./app/*` | TSX components, page routes, page-bound assets |
| `@lib/*` | `./src/*` | Domain logic in plain TS, no React |
| `@assets/*` | `./app/assets/*` | Bundled images Vite needs to fingerprint |

**Boundary rule:** `app/` may import from `src/` (via `@lib/*`). `src/` never imports from `app/`. If you need React-aware logic in `src/`, expose a hook (`use-*.ts`) and import it from `app/` — don't pull React state up.

## Build pipeline

```text
src + app + public
        │
        ▼
   tsc (--noEmit, pnpm check)   ← type errors gate everything else
        │
        ▼
   vite build                      ← bundles app/ + src/, copies public/
        │
        ▼
   dist/                           ← static SPA, ships to GitHub Pages
```

| Hook | Command | Purpose |
|------|---------|---------|
| `postinstall` | `node scripts/copy-pyodide.mjs` | Vendor Pyodide into `public/pyodide/` after `pnpm install` |
| `predev` | catalog + copy-pyodide | Catalog regenerated; Pyodide vendored before dev server starts |
| `prebuild` | catalog + copy-pyodide | Same, before production build |

## Component conventions

- Functional components only.
- `react-hooks/exhaustive-deps` is `error`-level — fix the dependency array, don't suppress.
- Keys must be stable. Don't use array index for keys when the list reorders.
- File >300 lines or component does >1 thing → split it.
- Every interactive element gets a `data-testid` so e2e tests don't depend on copy or DOM structure.
- Accessibility primitives (modals, menus, tooltips) come from `app/components/ui/` (shadcn over Radix). Don't roll your own.
